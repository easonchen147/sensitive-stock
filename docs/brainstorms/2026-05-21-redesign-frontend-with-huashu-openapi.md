# Huashu 前端全面升级 Brainstorm

## 背景

当前前端虽然已经替代旧默认入口，但整体仍是“迁移后工作台壳层 + 少数真实页面 + 多个 skeleton 页”的状态。用户要求本轮：

- 全部页面样式全面优化升级；
- 使用 `huashu-design` 的能力设计适合该系统的页面风格；
- 接入后端 `openapi.json` 覆盖的所有接口；
- 配合 OpenSpec + Compound 流程生成完整提案。

## 当前前端问题

### 1. 页面完成度分层明显

- 首页、回测页、市场页已有较明确的工作台形态。
- `screener`、`diagnosis` 仍然是诚实 placeholder。
- `factors`、`portfolio` 甚至还没有正式页面。

### 2. 风格还停在 phase-one 成熟壳层，而不是完整产品语言

这意味着：

- 有基础样式，但没有贯穿全站的信息层级体系。
- 缺少面向研究/操作工作台的统一页面模板。
- 缺少与后端契约对齐后的数据状态、空态、降级态、错误态系统。

### 3. 接口接入方式尚未 OpenAPI-first

当前前端已经有 BFF 代理与部分 helper，但还没有以统一 API 契约驱动：

- 类型生成
- endpoint 对齐
- 页面级数据装配

## Huashu 方向判断

对于这个系统，合适的不是营销站式 hero，而是：

- 研究工作台型信息建筑
- 高密度但有秩序的专业控制面板
- 清楚的状态语言与结果表达
- 对“行情 / 回测 / 诊断 / 因子 / 组合 / 选股”这些高信息负载场景的专门模板

也就是说，Huashu 在这里的价值不是“做一套炫视觉”，而是建立一套适合股票研究产品的全局页面语法。

## 推荐方向

### 风格定位

偏专业研究终端，而不是偏营销 SaaS：

- 克制但高级的配色与层级
- 强信息对比与扫描效率
- 统一的页面 band / 面板 / workbench 结构
- 数据状态、空态、错误态和结果解释有一致语言

### 页面覆盖范围

至少要覆盖：

- 登录页
- 首页 / cockpit
- 回测工作台
- 行情情报工作台
- 选股工作台
- AI 诊股工作台
- 因子分析工作台
- 组合优化工作台

## 关键依赖

- `complete-backend-openapi-platform` 先提供稳定 `openapi.json`。
- 半成品能力提案要先把后端接口与业务流程补齐。
- 回测替换提案决定回测页面的最终信息结构。

## 成功标准

- 前端所有主页面都不再是 placeholder / 临时壳层。
- 页面设计语言统一且适合研究系统。
- 前端接口接入以 OpenAPI 为事实来源。
- 新页面能够完整消费后端所有正式开放接口。
