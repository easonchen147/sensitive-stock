# 下一轮事件与风险增强候选

本轮已经把新闻和巨潮公告里的高价值事件抽取为 `eventHints`，并接入预测上下文、历史记录、前端展示和回测交接。下一轮继续优化时，优先考虑以下方向。

## 1. 公告正文解析

当前巨潮公告只使用结构化列表字段和公告标题，不解析 PDF 正文。下一步可以增加公告正文抓取与缓存，抽取金额、比例、期限、股东名称、问询事项和处罚类型等字段，让事件评分更精确。

建议验收点：

- `eventHints` 可以包含 `amount`、`ratio`、`deadline`、`counterparty` 等可选结构化字段。
- PDF 抓取失败时保留标题级事件提示，并返回降级原因。
- 不把 PDF 解析失败扩大为整个资讯接口失败。

## 2. 事件风险评分分层

当前事件评分是轻量规则分数。下一步可以把 `score` 拆成强度、可信度、时效、来源可靠性和重复压力，生成更可解释的 `riskLevel`。

建议验收点：

- 正负向事件都能输出 `riskLevel` 或 `impactLevel`。
- 重复转载不会线性放大事件强度。
- 官方公告、监管文件和普通媒体标题的可靠性权重不同。

## 3. 事件到行情的复盘闭环

当前预测历史支持按股票代码做最新行情方向评估。下一步可以把事件提示本身作为复盘对象，追踪事件后 1、3、5、10 个交易日的价格表现。

建议验收点：

- 每类事件可以统计历史样本数、平均收益、胜率、最大回撤。
- 前端市场页可以查看事件类型的历史表现摘要。
- 结果进入回测交接，支持一键验证某一类事件策略。

## 4. 事件冲突与组合解释

同一股票可能同时出现正向和负向事件。下一步可以增加冲突检测，例如“增持 + 监管问询”需要标记为混合信号，而不是简单按最高分事件输出单一方向。

建议验收点：

- 同一 `relatedSymbols` 同时命中正负事件时生成冲突提示。
- 本地启发式预测会降低冲突标的置信度。
- DeepSeek 上下文明确传入冲突事件组。

## 5. 事件订阅与告警

当前事件提示只在市场页被动展示。下一步可以增加订阅规则，例如关注股票、事件类型和负面风险阈值，并在接口层返回待处理告警。

建议验收点：

- 后端支持事件订阅配置和告警查询。
- 前端只展示真实后端告警，不新增空壳入口。
- 告警记录与预测历史可以互相跳转。
