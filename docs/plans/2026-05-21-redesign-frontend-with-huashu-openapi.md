# Huashu 前端全面升级 Plan

## 目标

以 Huashu 的设计方法为前端建立统一页面语言，并让所有页面基于 `openapi.json` 覆盖的接口完成接入。

## 设计方向

### 页面风格

采用“研究终端 / 专业工作台”方向：

- 高密度但清晰的层级
- 克制而明确的色彩与状态表达
- 针对研究分析工作流优化，而非营销式排版

### 信息结构

为每类页面建立统一模式：

- 头部摘要区
- 控制区 / 参数区
- 主结果区
- 辅助解释区
- 错误 / 空态 / degraded 态

### OpenAPI 集成

1. 基于 `openapi.json` 生成或对齐前端类型。
2. 统一 `frontend/lib` client 和 BFF route handler 行为。
3. 各页面按契约消费后端数据，不再手写漂移字段。

## 影响文件

- `frontend/app/*`
- `frontend/components/*`
- `frontend/lib/*`
- `frontend/types/*`
- 样式文件与设计 tokens

## 验证策略

- 页面级截图与布局检查
- 关键流程交互 smoke
- 所有页面的接口联调
- 样式系统一致性复核

## 里程碑

1. 明确全局设计方向与页面模板。
2. 统一首页 / 登录 / 导航壳层。
3. 重做回测 / 行情 / 选股 / 诊股 / 因子 / 组合六大工作台。
4. 全站完成 OpenAPI 驱动的接口接入。
